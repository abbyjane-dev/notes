    Github Accounts¶

    A non-exhaustive list of github users to follow

    🧱 Static sites, Jekyll, “boring but elegant” web¶

    GitHub Pages–adjacent, clean, sane people


    • @github/pages-gem

      • The canonical reference for what GitHub Pages actually supports. If something breaks locally vs Pages, this repo usually explains why.

    • @jekyll

      • Not flashy, but good signal on where Jekyll is going (and not going).

    • @mmistakes (Michael Rose)

      • Author of Minimal Mistakes theme, but more importantly: excellent examples of disciplined Jekyll structure and documentation.

    • @simonw (Simon Willison)

      • Static sites, personal publishing, data → HTML pipelines, zero BS. Very much your “small tools, sharp edges” vibe.

    📝 Documentation, writing, developer ergonomics¶

    People who treat docs as first-class


    • @danielecook

      • Thoughtful writing about docs, tooling, and long-lived systems.

    • @docwhat

      • Docs, accessibility, clarity, and technical writing culture.

    • @wilfred

      • Known for diff tooling and thoughtful posts on developer experience.

    • @karlhorky

      • Rails-y background, but very strong opinions about maintainable docs and tooling.

    🐍 Python (practical, not hype)¶

    • @kennethreitz

      • Requests, Pipenv, tooling that favors humans.

    • @tiangolo

      • FastAPI, excellent README hygiene, great examples of API documentation done right.

    • @nvbn

      • Small sharp Python tools, often very readable.

    🦀 Rust (approachable, not macho)¶

    Since you’ve been flirting with Rust but not in a “rewrite the world” way:


    • @burntsushi (Andrew Gallant)

      • Rust done responsibly. Deep explanations, zero hype, lots of text processing.

    • @dtolnay

      • Procedural macros, Serde internals — heavy stuff, but impeccably documented.

    • @matklad

      • Rust analyzer, thoughtful design writing, very “engineer’s engineer.”

    🧰 Unix, DevOps, infra minimalism¶

    People who hate unnecessary complexity:


    • @jpetazzo

      • Containers, infra history, thoughtful critiques of trends.

    • @iximiuz

      • Excellent explainer repos on Linux, containers, networking.

    • @leandromoreira

      • Computer architecture explainers that are actually readable.
